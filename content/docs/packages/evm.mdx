## Create Configuration

```ts title="lib/evm.config.ts" showLineNumbers
"use client";

import { create, EvmChains } from "@trezo/evm";

const contractAbi = [] as const;

export const evmConfig = create({
  address: "0x...",
  abi: contractAbi,
  chains: [EvmChains.optimismSepolia],
  rpcUrl: "https://optimism-sepolia-public.nodies.app",
  kit: {
    name: "connectkit",
    config: {
      projectId: process.env.NEXT_PUBLIC_WC_PROJECT_ID,
    },
  },
});

export const { Provider, ConnectButton } = evmConfig;
```

This configuration is the core of your app. It:

- Connects your contract (address + ABI)
- Defines supported chains
- Sets up wallet integration

It also generates:

- `Provider` â†’ wraps your app
- `ConnectButton` â†’ ready-to-use wallet UI
- `evmConfig()` â†’ hook for interacting with everything

## Read from Contract (`queryFn`)

```ts
const result = await call.queryFn("fetchAllTasks", []);
```

Use this for **view/pure functions**.

```ts
if (result.error) {
  console.error(result.error.message);
} else {
  console.log(result.data);
}
```

**Key points:**

- No wallet required
- No gas cost
- Fully typed from ABI
- Safe to call anytime

---

## Write to Contract (`mutateFn`)

```ts
const result = await call.mutateFn("addTask", ["New Task"]);
```

Use this for **state-changing functions**.

```ts
if (result.error) {
  console.error(result.error.message);
} else {
  console.log(result.data);
}
```

**Key points:**

- Requires wallet connection
- Prompts user signature
- Sends a transaction
- Returns transaction data

Always guard writes:

```ts
if (!wallet.account.isConnected) return;
```

---

## Listen to a Single Event (`listenFn`)

```ts
const unwatch = call.listenFn("TaskAdded", (...args) => {
  console.log("Task added:", args);
});
```

This subscribes to a contract event and runs every time it fires.

Cleanup is important:

```ts
unwatch();
```

**Use case:**

- Trigger refetch
- Update UI in real-time

---

## Listen to Multiple Events (`listenMany`)

```ts
const cleanup = call.listenMany({
  add: {
    eventName: "TaskAdded",
    listener: () => console.log("Added"),
  },
  update: {
    eventName: "TaskUpdated",
    listener: () => console.log("Updated"),
  },
});
```

Cleanup:

```ts
cleanup();
```

**Why this exists:**

- Avoid multiple `listenFn` calls
- Keep event logic centralized
- Cleaner for larger apps

## Error Handling Pattern

All calls follow the same structure:

```ts
const result = await call.queryFn("getTask", [1]);

if (result.error) {
  console.error(result.error.message);
  return;
}

console.log(result.data);
```

This avoids:

- try/catch everywhere
- inconsistent error shapes

---

## Recommended Pattern

Combine everything:

```ts
// 1. Read
await call.queryFn(...)

// 2. Write
await call.mutateFn(...)

// 3. React to blockchain updates
call.listenMany(...)
```

**Best practice:**

- Donâ€™t manually refetch after mutations
- Let events drive updates instead
