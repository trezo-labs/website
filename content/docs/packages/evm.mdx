## Install Package

```bash
npm install @trezo/evm
```

`@trezo/evm` gives you a **type-safe layer** for reading and writing to EVM smart contracts, connecting wallets, and reacting to on-chain events — all without having to wire up different packages or wallet providers yourself.

---

## Plugins Setup

<Steps>

### Vite Plugin

```ts title="vite.config.ts" showLineNumbers {3,6}
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import { trezoVitePlugin } from "@trezo/evm/plugins/vite";

export default defineConfig({
  plugins: [react(), trezoVitePlugin()],
});
```

Required for Vite apps. Deduplicates React so your app and the bundled wallet libraries share the same instance. Without it you'll get `"Invalid hook call"` errors. One-line fix, do it once.

</Steps>

## Create Configuration

<CodeCollapsibleWrapper>

```ts title="config/evm.config.ts" showLineNumbers
import { create, Chains } from "@trezo/evm";

const contractAbi = [...] as const;

export const config = create({
  address: "0x...",
  abi: contractAbi,
  chains: [Chains.optimismSepolia],
  rpcUrls: {
    11155420: "https://optimism-sepolia-public.nodies.app",
  },
  modalConfig: {
    from: "family", // "family" | "reown"
    options: {
      projectId: "your_wallet_connect_project_id",
      appInfo: {
        name: "My App",
      },
    },
  },
});

export const Provider = config.Provider;
export const ConnectButton = config.ConnectButton;
```

</CodeCollapsibleWrapper>

The **single source of truth** for your entire Web3 setup. It takes:

- **`address`** — deployed contract address
- **`abi`** — contract ABI typed `as const`. Without `as const` you lose all type safety.
- **`chains`** — supported chains. `Chains` re-exports all chains from `viem/chains`.
- **`rpcUrls`** — custom RPC endpoints keyed by chain ID. Avoids rate-limited public nodes.
- **`modalConfig`** — wallet modal provider. TypeScript autocompletes all supported options.
  - `"family"` — ConnectKit
  - `"reown"` — Reown AppKit

Returns a hook (`config()`) plus static properties (`Provider`, `ConnectButton`, `call`, `getState`, `subscribe`) usable anywhere — inside or outside React.

## Add Provider

```tsx showLineNumbers {1,6,8}
import { Provider } from "@/config/evm.config";
import App from "./App";

export default function Root() {
  return (
    <Provider>
      <App />
    </Provider>
  );
}
```

Must wrap your entire app. Sets up the wallet context, modal provider, and the internal store that bridges wallet state to contract calls. Without it, `config()` returns empty state and all calls run read-only with no signer.

## Basic Usage

<CodeCollapsibleWrapper>

```tsx title="App.tsx" showLineNumbers
"use client"; // required for Next.js

import { config, ConnectButton } from "@/config/evm.config";

export default function Home() {
  const { wallet, call, web3Provider } = config();

  return (
    <div>
      <ConnectButton>
        {({ isConnected, truncatedAddress, open, isConnecting }) => (
          <button onClick={() => open()} disabled={isConnecting}>
            {isConnected ? truncatedAddress : "Connect Wallet"}
          </button>
        )}
      </ConnectButton>

      {wallet.account.isConnected ? (
        <p>Connected: {wallet.account.address}</p>
      ) : (
        <p>Not connected</p>
      )}
    </div>
  );
}
```

</CodeCollapsibleWrapper>

`config()` is a React hook that gives you reactive access to wallet and contract state. Re-renders automatically when the wallet connects, disconnects, or the chain changes. Returns:

- **`wallet`** — connection state
- **`call`** — contract helpers (`queryFn`, `mutateFn`, `listenFn`)
- **`web3Provider`** — injected provider info

## ConnectButton

Renders wallet connection UI. Supports a render prop for full UI control, or a default button when no children are passed.

<Steps>

### Default button

```tsx
<ConnectButton label="Connect Wallet" />
```

### Custom render

```tsx
<ConnectButton>
  {({ isConnected, truncatedAddress, ensName, open, isConnecting }) => (
    <button onClick={() => open()}>
      {isConnected ? (ensName ?? truncatedAddress) : "Connect"}
    </button>
  )}
</ConnectButton>
```

### Opening specific views

```tsx
<ConnectButton>
  {({ isConnected, open, openAccountModal }) => (
    <button onClick={isConnected ? openAccountModal : () => open()}>
      {isConnected ? "Account" : "Connect"}
    </button>
  )}
</ConnectButton>
```

### Render props reference

| Prop               | Type                          | Description                      |
| ------------------ | ----------------------------- | -------------------------------- |
| `isConnected`      | `boolean`                     | Whether a wallet is connected    |
| `isConnecting`     | `boolean`                     | Whether the wallet is connecting |
| `address`          | `string \| undefined`         | Full wallet address              |
| `truncatedAddress` | `string \| undefined`         | Shortened e.g. `0x1234...5678`   |
| `ensName`          | `string \| null \| undefined` | ENS name if available            |
| `open`             | `(options?) => void`          | Opens the connect modal          |
| `close`            | `() => void`                  | Closes the modal                 |

</Steps>

## Read from Contract (`queryFn`)

For **view and pure** functions — reads that don't change state and cost no gas.

<Steps>

### Example

```ts
const result = await call.queryFn("fetchAllTasks", []);

if (result.error) {
  console.error(result.error.message); // human-readable, safe to show users
  console.error(result.error.raw); // full Error object for debugging
} else {
  console.log(result.data); // typed from your ABI
}
```

### Options

```ts
const result = await call.queryFn("fetchAllTasks", [], {
  from: "0x123...", // override sender (defaults to connected address)
  blockTag: "latest", // "latest" | "earliest" | "pending" | number
  gasLimit: 100000n,
  value: 0n,
});
```

### In a React component

```tsx
useEffect(() => {
  if (wallet.account.isConnecting) return; // wait for reconnect to finish

  call.queryFn("fetchAllTasks", []).then((result) => {
    if (!result.error) setTasks(result.data);
  });
}, [wallet.account.address, wallet.account.isConnecting]);
```

The `isConnecting` guard matters — on page reload wallets reconnect asynchronously. Fetching before reconnection completes means `msg.sender` is `address(0)` and you get empty results.

</Steps>

## Write to Contract (`mutateFn`)

For **nonpayable and payable** functions — writes that change on-chain state and require a wallet signature.

<Steps>

### Example

```ts
const result = await call.mutateFn("addTask", ["Buy groceries"]);

if (result.error) {
  toast.error(result.error.message); // e.g. "Content too long"
} else {
  console.log(result.data?.hash); // transaction hash
  console.log(result.data?.receipt); // full TransactionReceipt
}
```

Under the hood, `mutateFn`:

1. **Simulates** the transaction first via a direct RPC call, surfacing readable revert reasons before MetaMask opens.
2. **Sends** the real transaction through the connected wallet if simulation passes.
3. **Waits** for the transaction to mine and returns the receipt.

### Options

```ts
const result = await call.mutateFn("addTask", ["Buy groceries"], {
  estimateGas: true,
  gasLimit: 200000n,
  value: ethers.parseEther("0.01"),
  maxFeePerGas: ethers.parseUnits("20", "gwei"),
  maxPriorityFeePerGas: ethers.parseUnits("1", "gwei"),
  nonce: 42,
});
```

### Always guard writes

```ts
if (!wallet.account.isConnected) return;

const result = await call.mutateFn("addTask", ["Buy groceries"]);
```

</Steps>

## Listen to Events (`listenFn`)

Subscribes to contract events and fires your listener each time. Automatically falls back to polling if the RPC doesn't support WebSocket filters.

<Steps>

### Single event

```ts
const cleanup = call.listenFn("TaskAdded", (user, taskId, content) => {
  console.log("Task added:", { user, taskId, content });
});

cleanup(); // stop listening
```

Listener args are fully typed from your ABI.

### Multiple events

```ts
const cleanup = call.listenFn({
  add: { eventName: "TaskAdded", listener: fetchAllTasks },
  update: { eventName: "TaskUpdated", listener: fetchAllTasks },
  remove: { eventName: "TaskRemoved", listener: fetchAllTasks },
  toggle: { eventName: "TaskToggled", listener: fetchAllTasks },
});

cleanup(); // stops all listeners at once
```

More efficient than multiple `listenFn` calls — all listeners share the same underlying connection.

### In a React component

```tsx
useEffect(() => {
  return call.listenFn({
    add: { eventName: "TaskAdded", listener: fetchAllTasks },
    update: { eventName: "TaskUpdated", listener: fetchAllTasks },
    remove: { eventName: "TaskRemoved", listener: fetchAllTasks },
    toggle: { eventName: "TaskToggled", listener: fetchAllTasks },
  });
}, [wallet.account.address]); // re-register when wallet changes
```

The `wallet.account.address` dependency ensures listeners re-attach after wallet reconnection, preventing stale connections.

</Steps>

## Wallet State

```ts
const { wallet } = config();

wallet.account.isConnected; // true when wallet is connected
wallet.account.isConnecting; // true during connection or page-reload reconnect
wallet.account.address; // `0x${string}` | undefined
wallet.account.chainId; // number | undefined
```

Use `isConnecting` to differentiate between "not connected" and "reconnecting on reload":

```tsx
{
  wallet.account.isConnecting ? (
    <span>Reconnecting...</span>
  ) : wallet.account.isConnected ? (
    <span>{wallet.account.address}</span>
  ) : (
    <span>Not connected</span>
  );
}
```

## Provider State

```ts
const { web3Provider } = config();

web3Provider.isAvailable; // true if window.ethereum exists (MetaMask, etc.)
```

Use it to disable UI that requires an injected wallet:

```tsx
<button disabled={!web3Provider.isAvailable}>Add Task</button>
```

## Outside React (Vanilla JS)

Static methods for use outside React components — useful for utility functions or non-component files:

```ts
import { config } from "@/config/evm.config";

const state = config.getState();
console.log(state.wallet.address);

const result = await config.call.queryFn("fetchAllTasks", []);

const unsubscribe = config.subscribe(() => {
  console.log("State changed:", config.getState());
});
unsubscribe();
```

## Summary

| API                  | Purpose                                          |
| -------------------- | ------------------------------------------------ |
| `create(config)`     | Initialize the entire Web3 setup                 |
| `Provider`           | App wrapper — required                           |
| `ConnectButton`      | Wallet connection UI with render props           |
| `config()`           | React hook — access wallet, call, provider state |
| `call.queryFn`       | Read contract data (view/pure functions)         |
| `call.mutateFn`      | Write to contract (nonpayable/payable functions) |
| `call.listenFn`      | Single or multiple event listeners               |
| `wallet`             | Connection state (address, chainId, isConnected) |
| `web3Provider`       | Injected provider availability                   |
| `config.getState()`  | Static access outside React                      |
| `config.subscribe()` | Subscribe to state changes outside React         |
| `trezoVitePlugin()`  | Vite plugin — prevents duplicate React           |
