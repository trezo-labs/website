## Overview

The **Trezo CLI** is the fastest way to get started with building multi-chain Web3 applications.

Instead of manually setting up dependencies, wallet integrations, and blockchain configurations, the CLI handles everything for you by:

- Fetching **curated templates** from a remote registry
- Cloning production-ready starters into your local environment
- Guiding you through a simple, interactive setup process

This ensures that every project follows a **consistent structure and developer experience**, regardless of the blockchain or framework you choose.

---

## Why Use Trezo CLI?

- **Zero setup friction** — no need to configure Web3 providers manually
- **Multi-chain ready** — build across different ecosystems using a unified pattern
- **Framework flexibility** — choose from templates like Vite, Next.js, and more
- **Opinionated structure** — best practices built-in from the start

## Initialize a Project

```bash
npx trezo init
```

You can run the CLI directly without installing it globally.

This command starts an interactive setup that walks you through creating your project step-by-step.

---

## Commands

<Steps>

### **trezo init**

Initialize a new Trezo-powered Web3 project.

```bash
trezo init [options]
```

#### What It Does

- Prompts you to select a blockchain package
- Lets you choose a starter template
- Clones the selected template into your local machine
- Prepares your project with all required dependencies and configurations

---

#### Options

- `-n, --name <name>`
  Specify the project name (and directory). If omitted, you will be prompted.

- `-p, --package <name>`
  Skip package selection and directly use a specific package (e.g. `@trezo/evm`).

- `-t, --template <name>`
  Skip template selection and use a predefined template (e.g. `vite-app`).

---

#### Interactive Flow

When no flags are provided, the CLI guides you through:

1. **Package Selection**
   Choose the blockchain ecosystem you want to build on.

2. **Template Selection**
   Pick a starter template (e.g., Vite, Next.js).

3. **Project Naming**
   Confirm or customize your project directory.

4. **Project Scaffolding**
   The selected template is cloned and configured automatically.

---

#### Example Usage

```bash
# Fully interactive
trezo init

# Skip prompts using flags
trezo init -n my-dapp -p @trezo/evm -t vite-app

# Full explicit usage
trezo init --name my-dapp --package @trezo/evm --template vite-app
```

---

#### After Setup

Once your project is created, the CLI will display recommended next steps such as:

```bash
pnpm install
pnpm dev
```

These commands are defined by the selected template and should be run manually.

---

### **trezo list**

View all available blockchain packages and templates.

```bash
npx trezo list
```

#### What It Shows

- Available Trezo packages (e.g., EVM, Starknet)
- Supported templates under each package
- Template identifiers for CLI usage

---

#### Example Output

```txt
📦 @trezo/evm – Ethereum { package: evm }
  ├─ Next.js → evm-next-js
  └─ Vite → evm-vite

📦 @trezo/strk – Starknet { package: strk }
  └─ Starknet React → strk-react

Found 2 package(s)

Usage: trezo init -p <package> -t <template>
```

---

#### When to Use

Use `trezo list` when you:

- Want to explore available templates
- Need exact package or template IDs
- Prefer non-interactive CLI usage
